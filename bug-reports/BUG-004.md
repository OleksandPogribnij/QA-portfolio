# [BUG-004] Ability to log in using unregistered data

## General information
* **Project:** [PetStore API](https://petstore.swagger.io/);
* **Priority:** Highest;
* **Severity:** Blocker;
* **Type:** Security Bug;
* **Status:** Open;

---

## Environment
* **Base URI:** https://api.petstore.com/v2;
* **Endpoint:** /user/login (GET);
* **Tool:** Postman v11.81.4;

---

## Description
The system allows authentication via the POST /login API request using arbitrary (non-existent in the database) credentials. The backend returns a **200 OK** status code, granting access to authorized user functions.

### Steps to Reproduce
1. Execute a GET request for login using the following test data (see Test Data)

### Test Data
```json
{
  "username": "non-existent-username",
  "password": "non-existent-pass"
}
```

### Actual Result
The server successfully processes an authentication request with non-existent or random credentials:
- **Status Code**: 200 OK (instead of the expected error).
- **Response Body**: The server returns the message “logged in user session:1770420132868” in the “message” field.
- **Behavior**: The system ignores the absence of the user in the database and creates an active session for the non-existent profile.

### Expected Result
The system must strictly verify the user's presence in the database and password matching before granting access.
