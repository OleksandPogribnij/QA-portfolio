# [BUG-001] 500 status code when sending incorrect data types

## General information
* **Project:** [PetStore API](https://petstore.swagger.io/);
* **Priority:** Medium;
* **Severity:** High;
* **Type:** Validation Error;
* **Status:** Open;

---

## Environment
* **Base URI:** https://api.petstore.com/v2;
* **Endpoint:** /user (POST);
* **Tool:** Postman v11.81.4;

---

## Description
When executing a POST request to create a user (/user), the backend does not perform proper data type validation. When a string is passed to fields that expect an integer (e.g., id or userStatus), the server returns a 500 Internal Server Error status code instead of a correct client validation error.

### Steps to Reproduce
1. Execute a POST request with the following (invalid) test data (see Test Data block);

### Test Data
```json
{
  "id": "string",
  "username": 0,
  "firstName": 0,
  "lastName": 0,
  "email": 0,
  "password": 0,
  "phone": 0,
  "userStatus": "string" 
}
```

### Actual Result
In response, we receive a **500 Internal Server Error** status code, indicating that there is no data validation on the backend.

### Expected Result
In response, we receive a **400 Bad Request** status code, which indicates that data validation is performed on the backend.
